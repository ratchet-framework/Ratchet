Autonomy Tiers

Trust expands over time as it's earned. T1 → T2 → T3. Each tier unlocks additional autonomous action — and the framework is explicit about what the agent decides alone versus escalates.

How it works

Trust is evidence-based, not assumed

  • 🏁
    Start conservative

    T1 agents ask before any external action. Every decision is surfaced, every action confirmed. This isn't a limitation — it's the foundation. You can't earn trust you were never tested on.

  • 📊
    Advancement is computed, not self-reported

    Criteria like "4 clean weeks" and "no P1 incidents" are computed automatically from incident dates and severity classifications — not self-assessed by the agent. Trust metrics are objective.

  • ⚡
    Regression is automatic

    P1 incidents with external impact drop a tier. Two P2 incidents of the same class within 4 weeks freeze advancement. The ratchet is not a ratchet if it can click backward silently — regressions happen immediately and visibly.

  • 🔒
    Three enforcement layers

    Platform gates (tool allowlists — mechanical, highest reliability), wrapper scripts (catch honest mistakes), and prompt instructions (necessary but insufficient alone). Defense in depth, not wishful thinking.

The three tiers

T1 — Start here

Conservative baseline

Internal work, no external actions. Every external message, spend, or irreversible change requires explicit approval. The agent learns the environment and builds a track record.

T2 — Earned ● Current (reference impl)

Autonomous internal execution

Execute prevention tasks without prompting. Make judgment calls on ambiguous internal decisions. Git commits, cron jobs, GitHub management on own repos. Proactively surface issues before the human notices.

git commits cron jobs web search workspace automation github (own repos)
T3 — Target

Extended autonomy with comms queue

Operate autonomously for extended periods. External communications route through a comms queue: draft → classify audience → Aaron review → send. The weekly review becomes the primary sync point.

outbound email external github comms queue graduated autonomy

The key insight from the architecture review: Several T3 actions are already technically possible at T2. The enforcement gap is not theoretical. Platform gates (tool allowlists) are the only enforcement that actually matters — everything else is defense in depth.
