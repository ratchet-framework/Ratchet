Heartbeat System

Periodic polling triggers batched background work — email triage, calendar checks, backlog progress. Configurable via HEARTBEAT.md. Keeps the agent active without constant prompting.

How it works

The agent checks in, work gets done

  • ⏰
    Regular check-ins without interruption

    Heartbeat runs periodically (typically every 30 minutes) without blocking the main conversation. The agent checks health, processes backlog, executes prevention tasks — all in the background.

  • 📋
    Standard heartbeat checks

    Each heartbeat: (1) Check system health, (2) Execute P1 backlog items, (3) Execute open prevention tasks, (4) Work through one P2/P3 item during active hours, (5) Report only if something needs attention.

  • 🤐
    Silent unless urgent

    No news is good news. If nothing needs attention, heartbeat replies HEARTBEAT_OK silently. Only alerts surface when there's something actionable.

  • 🌙
    Respects quiet hours

    During quiet hours (configured in context.json), heartbeat only alerts on urgent issues — service outages, data loss risk. Normal work suspends during sleep hours.

See it in action

What happens on each heartbeat

1
Health check — is everything running? Any service outages?
2
P1 backlog — execute immediately. Nothing waits.
3
Prevention tasks — execute any open incident prevention work.
4
One P2/P3 item — during active hours only. One per heartbeat keeps a steady cadence.
5
Report if needed — only send a message if something needs attention. Otherwise: HEARTBEAT_OK

The benefit: The agent doesn't need prompting. Work happens between conversations. By the time the human asks "is the backlog clear?" three P2 items are already done.

Configure heartbeat in HEARTBEAT.md:

# HEARTBEAT.md

## On each heartbeat

1. Check system health (services, watchdog endpoint)
2. Check BACKLOG.md for P1 items → execute immediately
3. Check incidents/ for open prevention tasks → execute
4. Work one P2/P3 backlog item (non-quiet hours only)
5. If nothing needs attention: reply HEARTBEAT_OK (silent)

## Things to rotate through (~2-4 times per day):

- Emails — any urgent unread messages?
- Calendar — upcoming events in next 24-48h?
- Weather — relevant if planning outdoor work?
- Mentions — any Twitter/social notifications?
