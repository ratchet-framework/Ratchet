Platform Adapters

Ratchet is conventions, not code. Wire it to OpenClaw, Claude Code, Cursor, or any agent platform. Write an adapter, contribute it back. Reference implementation ships with OpenClaw.

How it works

The Ratchet stack is platform-agnostic

  • 📁
    Portable by design

    Ratchet is a collection of files and conventions — MEMORY.md, BACKLOG.md, incidents/, memory/, context.json. No database, no API, no proprietary platform. Copy the files to your workspace, wire the hooks, you're done.

  • 🪝
    Three platform hooks

    on_session_end → run memory extraction. on_session_start → inject relevant facts. on_weekly_review → run memory management. Any platform that supports these hooks can use Ratchet.

  • 🔧
    Reference implementation: OpenClaw

    OpenClaw is the proven platform for Ratchet. Full integration with heartbeat, cron jobs, context loading, and Telegram delivery. If you want to run Ratchet today, OpenClaw is the fastest path.

  • 🤝
    Contribute your adapter

    Built Ratchet on Claude Code? Cursor? LLaMA? Write an adapter, document how to wire it up, and submit a PR. The framework is only as useful as the platforms it runs on.

Your platform?

Ratchet is designed to be portable. If your agent platform has file I/O and can execute scripts, Ratchet works.

The key insight: Ratchet doesn't depend on any specific platform. By using flat files, it stays portable across Claude Code, Cursor, raw LLMs, and custom agent frameworks. Your agent data isn't locked into a proprietary system.

Building an adapter

Three required hooks:

| Hook | Trigger | Action |
|------|---------|--------|
| on_session_end | Before context compaction | Run bin/memory-extract |
| on_session_start | After context loads | Run bin/memory-retrieve, inject facts |
| on_weekly_review | Friday review cron | Run bin/memory-manage |
