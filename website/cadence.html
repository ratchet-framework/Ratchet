Cadence Tracking

Track anything with a maintenance interval — by time, usage, or both. Pre-service checklists in every alert. Log a completed service in one message. Works for vehicles, infrastructure, habits.

How it works

Both dimensions. Whichever threshold hits first.

  • 📅
    Time + usage simultaneously

    Calendar reminders fire on fixed dates. Real-world maintenance doesn't work that way. An oil change is due at 5,000 miles or 6 months — whichever comes first. Cadence tracks both and projects when you'll hit each threshold.

  • 🔔
    Alert window, not just due date

    Each item has an alert window — warn me 2 weeks or 500 miles before I hit the threshold. Items in the window surface in the daily briefing. Overdue items surface in every briefing until resolved.

  • 📈
    Usage rate projection

    The agent calculates typical usage rate from history and projects arrival at threshold. "At your current rate of ~250 miles/week, WRX oil change is due in approximately 18 days" — not just "due in 300 miles."

  • ✅
    Log completion in one message

    When maintenance is done: "logged oil change at 11,800 miles." The agent identifies the cadence item, runs the update, and confirms: "Logged. Next oil change due ~17,800 miles or August 2026." No forms, no menus.

The problem it solves: Calendar reminders fire on fixed dates. Real-world maintenance doesn't work that way. If you drove hard this month, the calendar reminder is wrong. If the car sat all winter, the mileage reminder is wrong. Cadence tracks both dimensions simultaneously.

Data model (cadence.json):

{
  "id": "wrx-oil",
  "label": "WRX oil change",
  "category": "vehicle",
  "lastServiceDate": "2025-09-01",
  "lastServiceUsage": 9800,
  "currentUsage": 11760,
  "usageUnit": "miles",
  "thresholds": {
    "time":  { "value": 6, "unit": "months" },
    "usage": { "value": 5000, "unit": "miles" }
  },
  "alertWindow": {
    "time":  { "value": 2, "unit": "weeks" },
    "usage": { "value": 500, "unit": "miles" }
  }
}
