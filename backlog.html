Autonomous Backlog

Self-identified tasks execute during heartbeats. P1s run immediately. P2/P3 work through the backlog without prompting. The agent surfaces its own work.

How it works

The agent's self-directed work queue

  • 🎯
    Three priority levels, one file

    P1 items run on the next heartbeat — no waiting. P2 items ship within the week. P3 items work through on available bandwidth. The priority system is the scheduling system: there's no separate scheduler.

  • 🔄
    Items enter from multiple sources

    Incident prevention tasks, gap audits (things the agent notices it can't do well), and strategic improvements all flow into the same backlog. Unified queue, unified execution model.

  • 💓
    Heartbeat-driven execution

    Every heartbeat checks the backlog. P1 items execute immediately. One P2/P3 item executes per heartbeat during non-quiet hours. Work happens automatically, between conversations — the agent doesn't need prompting.

  • 📊
    Closed items stay visible

    Completed items move to a Completed table with the outcome noted. This creates a record of what the agent accomplished autonomously — useful for weekly reviews and trust tier advancement evidence.

Priority levels

P1 / P2 / P3

P1
Do immediately
Next heartbeat. No exceptions. Typically: incident prevention tasks, active service outages, anything with external impact.
P2
This week
Work through during non-quiet hours. One item per heartbeat. Typically: capability improvements, process gaps, medium-priority prevention tasks.
P3
When bandwidth allows
Low urgency, real value. Nice-to-haves, documentation, optimizations. Gets done when nothing else is pressing.
